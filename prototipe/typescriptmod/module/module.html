Conditional Types

Los tipos condicionales en TypeScript permiten definir tipos en función de una condición dada. Son útiles para crear tipos más flexibles y reutilizables.

Ejemplo básico

En este ejemplo, determinamos si un tipo es una cadena o no.

type IsString<T> = T extends string ? "Es un string" : "No es un string";

En este caso, IsString es un tipo condicional que verifica si T es una cadena. Si lo es, devuelve "Es un string", de lo contrario, devuelve "No es un string".

Uso con genéricos

Podemos inferir el tipo dentro de un array usando la palabra clave infer.

type InferType<T> = T extends Array<infer U> ? U : never;

El tipo InferType utiliza infer para extraer el tipo de los elementos de un array. Si T es un array, U será el tipo de sus elementos, de lo contrario, será never.

Restricción en tipos condicionales

Este ejemplo verifica si un tipo es un número o no.

type CheckNumber<T> = T extends number ? "Es un número" : "No es un número";

El tipo CheckNumber comprueba si T es un número. Si lo es, devuelve "Es un número", de lo contrario, devuelve "No es un número".

Combinar condicionales

Podemos combinar múltiples condiciones en un solo tipo condicional.

type MultiCheck<T> = T extends string ? "Es un string" : T extends number ? "Es un número" : "Otro tipo";

El tipo MultiCheck verifica si T es una cadena, un número o cualquier otro tipo, devolviendo un mensaje apropiado para cada caso.

Ejemplo avanzado

Aquí verificamos si una interfaz tiene un campo id y retornamos un mensaje apropiado.

type ConditionalExample<T> = T extends { id: number } ? "Tiene ID" : "No tiene ID";

El tipo ConditionalExample comprueba si T tiene una propiedad id de tipo número. Si es así, devuelve "Tiene ID", de lo contrario, devuelve "No tiene ID".

Tipos condicionales distribuidos

Cuando se utilizan tipos condicionales con un tipo genérico que puede ser una unión, TypeScript distribuye el tipo condicional sobre cada miembro de la unión.

type Distributed<T> = T extends string ? "Es un string" : "No es un string"; type Test = Distributed; // "Es un string" | "No es un string"

En este ejemplo, Distributed se aplica a cada miembro de la unión string | number, resultando en una unión de los resultados.

Tipos condicionales recursivos

Los tipos condicionales también pueden ser recursivos, lo que permite definir tipos complejos y anidados.

type Flatten <T> = T extends Array ? Flatten<U> : T; type Example = Flatten; // number

El tipo Flatten recursivamente aplana un array de arrays hasta obtener el tipo base.
